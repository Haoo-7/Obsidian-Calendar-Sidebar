# On This Day (去年今日) — 功能设计

**日期**: 2026-07-18
**状态**: 已实现 (v1.2.0)
**插件**: calendar-sidebar/main.js

## 概述

在 Calendar Sidebar 中新增"去年今日"回顾功能。用户可以看到往年同一日期（月-日）的日记照片和摘要，并通过日历格子上的小标记快速发现哪些日期有往年记录。

## 功能组成

### C: 日历格子标记

- 位置: 日期格子右下角
- 样式: 小圆点 (直径 6px)，颜色使用 `var(--color-accent)`
- 触发: 该日期存在往年（任意年份）的同月同日日记
- 设置开关: `showOnThisDayDot` (boolean, 默认 true)
- 实现: `OnThisDayProvider` 在渲染日历时批量查询当月所有日期的往年数据，标记结果缓存到当月末

### B: 回顾弹窗

#### 入口

两个入口，各自可配置开关:

1. 侧边栏日历面板底部按钮 `showOnThisDayButton` (默认 true)
2. 命令面板命令 "Open On This Day / 打开去年今日"

按钮放在天气卡片下方，显示当前日期（如 "📅 7月18日"）。点击打开弹窗。如果今天没有历史记录，按钮变灰/隐藏（取决于有无数据）。

#### 弹窗布局

```
┌──────────────────────────────────┐
│  📅 去年今日 — 7月18日       ✕  │  ← 标题栏，毛玻璃
├──────────────────────────────────┤
│                                  │
│  ┌──────────────────────────┐   │
│  │       2025年             │   │  ← 年份标签
│  │  ┌──────────────────┐   │   │
│  │  │                  │   │   │
│  │  │   [照片/图片]     │   │   │  ← 日记首图，cover填充
│  │  │                  │   │   │
│  │  └──────────────────┘   │   │
│  │  "今天去了图书馆，读完   │   │  ← 摘要（最多3行）
│  │   了那本宋史..."        │   │
│  └──────────────────────────┘   │
│                                  │
│    [ ← 上一年 ]  1/2  [ → ]     │  ← 翻页导航
│                                  │
└──────────────────────────────────┘
```

- 宽度: 420px (桌面端) / 90vw (移动端)
- 图片最大高度: 280px, cover + 顶部对齐
- 无图年份: 显示空状态 "这一天还没有记录"
- 所有年份都没有: 显示 "还没有往年的今天" 并自动关闭
- 点击图片或摘要区域 → `workspace.openLinkText()` 打开对应日记
- 点击弹窗背景/关闭按钮 → 关闭
- 按 Esc → 关闭
- 按 ← → 方向键 → 翻年份

#### 数据提供

新增 `OnThisDayProvider`:

```js
getEntries(month, day): [{ year, file: TFile, images: string[], excerpt: string }]
```

- 扫描 `dailyFolder` 下所有 `YYYY-MM-DD.md` 文件（按命名规则匹配）
- 使用 `metadataCache.getCache()` 获取嵌入图片（复用现有 `_extractImagePaths`）
- 按年份降序排列（最近的在前）
- 按 (月, 日) 缓存结果
- 文件变更时通过现有 `_onFileChanged` 钩子失效缓存

#### 摘要提取

三种模式，设置项 `excerptMode` (默认 "auto"):

| 值 | 说明 |
|----|------|
| `"frontmatter"` | 从 frontmatter 的 `excerpt` 字段读取（用户可自定义 key） |
| `"auto"` | 自动提取正文纯文本前 100 字 |
| `"none"` | 不显示摘要，弹窗仅展示图片 |

自动提取逻辑:
1. 读文件内容，剥掉 YAML frontmatter
2. 去掉 `![[...]]` 和 `[[...]]` wiki 链接语法
3. 去掉 Markdown 标题标记 `#`
4. 去掉 `*` `_` `~` `==` 等格式标记
5. 合并多余空白，取前 100 字
6. 如果正文为空 → 显示 "（无文字内容）"

### 设置项汇总

| Key | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| `showOnThisDayDot` | boolean | true | 日历格子上显示往年标记 |
| `showOnThisDayButton` | boolean | true | 侧边栏显示回顾入口按钮 |
| `excerptMode` | string | "auto" | 摘要提取方式 |
| `excerptFrontmatterKey` | string | "excerpt" | frontmatter 模式下读取的字段名 |

## 性能

- 月历渲染时: 预查询当月 31 天的往年数据 → 31 次文件扫描。目前 2 年数据量，几乎无感。后续年份多了可加并发限制。
- 弹窗打开时: 只需查当前日期的数据，大部分已被预缓存。
- 缓存: 按 (月, 日) 缓存，月切换时自动失效旧缓存。
